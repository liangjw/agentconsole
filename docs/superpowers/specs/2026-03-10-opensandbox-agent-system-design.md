# OpenSandbox 智能代理系统设计

## 1. 项目概述

- **项目名称**: AgentConsole
- **项目类型**: Web应用 (SaaS)
- **核心功能**: 基于OpenSandbox沙箱的智能代理创建与任务执行平台
- **目标用户**: 产品经理和非技术背景人员

## 2. 技术栈

- **前端**: Next.js 14+ (App Router)
- **后端**: FastAPI
- **沙箱引擎**: OpenSandbox
- **数据库**: SQLite (开发) / PostgreSQL (生产)
- **部署**: Docker Compose

## 4. 数据模型

### 4.1 Agent (智能代理)

| 字段 | 类型 | 描述 |
|------|------|------|
| id | UUID | 主键 |
| name | String | 代理名称 |
| description | String | 代理描述 |
| skills | JSON | 技能配置列表 |
| image | String | 沙箱镜像配置 |
| created_at | DateTime | 创建时间 |
| updated_at | DateTime | 更新时间 |

### 4.2 Skill (技能)

| 字段 | 类型 | 描述 |
|------|------|------|
| id | UUID | 主键 |
| name | String | 技能名称 |
| description | String | 技能描述 |
| type | String | 工具类型 |
| definition | JSON | 技能定义(schema) |
| icon | String | 图标 |

### 4.3 Conversation (会话)

| 字段 | 类型 | 描述 |
|------|------|------|
| id | UUID | 主键 |
| agent_id | UUID | 关联的代理 |
| user_id | String | 用户ID |
| sandbox_id | String | 沙箱实例ID |
| status | String | 状态: active/completed |
| created_at | DateTime | 创建时间 |

### 4.4 Message (消息)

| 字段 | 类型 | 描述 |
|------|------|------|
| id | UUID | 主键 |
| conversation_id | UUID | 关联的会话 |
| role | String | 角色: user/assistant |
| content | String | 消息内容 |
| tool_calls | JSON | 工具调用记录 |
| created_at | DateTime | 创建时间 |

## 5. API 设计

### 5.1 代理管理

| 方法 | 路径 | 描述 |
|------|------|------|
| POST | /api/agents | 创建代理 |
| GET | /api/agents | 获取代理列表 |
| GET | /api/agents/{id} | 获取代理详情 |
| PUT | /api/agents/{id} | 更新代理 |
| DELETE | /api/agents/{id} | 删除代理 |

### 5.2 会话管理

| 方法 | 路径 | 描述 |
|------|------|------|
| POST | /api/conversations | 创建会话(启动沙箱) |
| GET | /api/conversations/{id} | 获取会话 |
| POST | /api/conversations/{id}/messages | 发送消息 |
| DELETE | /api/conversations/{id} | 结束会话(销毁沙箱) |

### 5.3 Skills管理

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | /api/skills | 获取可用Skills列表 |
| POST | /api/skills | 添加自定义Skill |

## 6. 前端页面

### 6.1 代理列表页 (`/agents`)
- 显示用户创建的所有代理
- 创建新代理按钮
- 每个代理显示: 名称、描述、Skills数量
- 编辑/删除操作

### 6.2 代理创建/编辑页 (`/agents/new`, `/agents/{id}/edit`)
- 代理名称输入
- 代理描述输入
- Skills多选组件
- 镜像配置下拉选择
- 保存/取消按钮

### 6.3 会话页 (`/conversations/{id}`)
- 聊天消息列表
- 用户输入框
- 发送按钮
- 结束会话按钮
- 代理思考中状态显示

## 7. 核心流程

### 7.1 创建代理流程
1. 用户填写代理名称、描述
2. 选择需要的Skills
3. 选择沙箱镜像
4. 保存代理配置

### 7.2 会话执行流程
1. 用户选择代理并启动
2. 后端创建OpenSandbox沙箱实例
3. 将配置的Skills加载到沙箱工作空间
4. 用户发送自然语言任务
5. 代理理解任务，在沙箱中执行操作
6. 返回执行结果给用户
7. 用户可继续交互或结束会话
8. 结束时会话自动销毁沙箱实例

## 8. 安全考虑

- 沙箱完全隔离，用户无法直接访问
- 所有文件操作通过代理完成
- 沙箱网络访问可控
- 会话超时自动清理

## 9. 部署配置

### Docker Compose
- `nextjs` 服务: 前端
- `fastapi` 服务: 后端
- `opensandbox` 服务: 沙箱引擎
